# DATA_SPEC

## 入力フォーマット

入力は `posts.md` という Markdown / text ファイルを前提とする。

### 形式
- 区切り行 `---` をセパレータとして使う
- `---` に挟まれた本文1つを1投稿とみなす
- 本文中の改行は保持する
- 空ブロックは無視する

### 例
```md
---
投稿文1
---
投稿文2
2行目
---
投稿文3
---
パースルール

行単位で読み込む

--- の単独行をセパレータとみなす

セパレータ間の文字列を1投稿として抽出する

前後の空白行は trim する

空文字になったブロックは除外する

スケジュール生成ルール

1件目を 07:00 に割り当てる

以後は1時間ごとに割り当てる

同日最後は 22:00

1日最大 16件

17件目以降は翌日の 07:00 から再開する

タイムゾーンは Asia/Tokyo

既に過ぎた当日枠はスキップして次の有効枠へ割り当てる

内部型
type RawPost = {
  text: string;
  order: number;
};

type ScheduledPost = {
  id: string;
  text: string;
  order: number;
  scheduledAt: string;
};
バリデーション

投稿数が0件ならエラー

各投稿本文が空なら除外

長すぎる投稿はエラー

区切りのみで本文がないブロックは無視する
