# AGENTS.md

このリポジトリで作業するAIコーディングエージェント向けの指示。

## プロジェクトの本質
このアプリは、X公式Web UI上で「予約投稿を保存する」ためのローカル自動化ツールである。
SNSエンゲージメントを人工的に増加させる機能は実装しない。

## 最優先事項
1. 安定性
2. 安全な停止
3. 可読性
4. 拡張性
5. 実装速度

## 絶対にやってはいけないこと
- 自動フォロー機能の追加
- 自動いいね機能の追加
- 自動リポスト機能の追加
- 自動返信やDM機能の追加
- CAPTCHA突破処理の追加
- 電話認証や本人確認の回避処理
- 複数アカウントを前提にした設計
- ステルス化や検知回避を目的とする実装
- 不自然な高速連打

## 実装方針
- TypeScript を使用する
- UI操作は Playwright を使用する
- headful モードをデフォルトにする
- locator は堅牢性重視で実装する
- data-testid 優先、なければ role / label / text を使う
- セレクタは1箇所に集約する
- 失敗時は screenshot を保存する
- ログは構造化する
- 認証状態は storageState で再利用する
- 実行前バリデーションを厳密に行う

## コード品質ルール
- any を極力使わない
- 小さな関数に分割する
- 1ファイル1責務を意識する
- コメントは「なぜ」を書く
- 例外は握り潰さない
- エラーメッセージはユーザーが理解できる文言にする

## 推奨実装順
1. データ型定義
2. ファイル読み込み
3. スケジュール生成
4. CLI引数処理
5. Xログイン状態読込
6. 1件分の予約保存
7. 複数件バッチ処理
8. ログ出力
9. dry-run
10. テスト

## 完了条件
- md から投稿データを読める
- 7:00〜22:00の1時間間隔で予約枠を生成できる
- Xで1件以上予約保存できる
- エラー時に停止・記録できる
- dry-run が動作する
- README の手順で再現できる
