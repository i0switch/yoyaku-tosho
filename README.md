# X Scheduler Builder

Xの公式Web UIを利用して、Markdownファイルから予約投稿を生成・保存するローカル自動化ツール。

## セットアップ

1. 依存関係のインストール:
   ```bash
   npm install
   ```
2. ブラウザのインストール:
   ```bash
   npx playwright install chromium
   ```
3. 認証情報の作成:
   下記コマンドを実行するとブラウザが開くから、ログインしてね。
   ホーム画面が表示されたらブラウザを閉じれば、セッションが保存されるよ。
   ```bash
   npm run setup-auth
   ```

### ログインがはじかれる場合 (検知回避)
通常の自動操作ブラウザだとXにログインできない場合があります。その場合は、普段使っているChromeをデバッグモードで起動して接続する方法が有効です。

1. Chromeをデバッグモードで起動:
   ```bash
   npm run chrome-debug
   ```
2. 立ち上がったブラウザでXにログイン。
3. アプリを実行（`--cdp` オプションを使用）:
   ```bash
   npm start -- --cdp http://localhost:9222
   ```

## 使い方

### 投稿データの作成
`posts.md` を作成し、`---` で投稿を区切ります。

```md
---
投稿1の内容
---
投稿2の内容
---
```

### 実行

**Dry-run (実行計画の確認のみ)**:
```bash
npm start -- --dry-run
```

**本番実行 (実際にブラウザで操作)**:
```bash
npm start
```

### オプション
- `-i, --input <path>`: 入力ファイル名 (デフォルト: `posts.md`)
- `-s, --storage <path>`: セッション保存ファイル (デフォルト: `storageState.json`)
- `-m, --max <n>`: 最大処理件数
- `--headless true`: ヘッドレスモードで実行 (デフォルト: `false`)

## Windowsユーザー向け（GUIモード）

`launch.bat` をダブルクリックするとブラウザでGUI管理画面が開くよ。
コマンド操作が難しい場合はこっちを使ってね。

（コンソールウィンドウを出したくない場合は `launch.vbs` をダブルクリック）

---

## ⚠️ storageState.json の取り扱い

`npm run setup-auth` を実行すると `storageState.json` が生成される。
このファイルには **Xのログインセッション情報（認証トークン等）** が含まれる。

- 他人と共有しない
- GitHubなどに公開しない（`.gitignore` で除外済みだけど要注意）
- 流出すると第三者にアカウントを操作される可能性がある
