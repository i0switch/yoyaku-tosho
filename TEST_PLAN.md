# TEST_PLAN

## テスト方針
UI依存があるため、ロジック部分は単体テストを厚くし、X UI 部分は最小限のE2Eで確認する。

## 1. 単体テスト対象

### スケジュール生成
- 投稿数 1件
- 投稿数 16件
- 投稿数 17件
- 現在時刻が 06:00
- 現在時刻が 07:30
- 現在時刻が 22:30
- 個別日時指定あり
- 日跨ぎあり

### 入力バリデーション
- text 欠落
- 空文字
- mediaPath 不存在
- date 形式異常
- time 形式異常
- CSV 空行
- JSON 配列でない

### CLI引数
- 入力ファイル未指定
- 不正パス
- max が 0
- offset が負数

## 2. 結合テスト対象
- CSV 読込 → スケジュール生成 → dry-run 出力
- JSON 読込 → スケジュール生成 → dry-run 出力
- mediaPath ありの投稿を含むケース

## 3. E2Eテスト対象
- auth state 読込済みで X 投稿画面へ遷移
- 1件の予約保存
- 画像付き1件の予約保存
- 予約ダイアログ未表示時の異常停止
- セレクタ取得失敗時のスクリーンショット保存

## 4. 手動確認項目
- 予約投稿一覧に保存されていること
- 文字化けしないこと
- 改行が維持されること
- 画像添付が正しいこと
- ログが十分読めること

## 5. 受け入れ基準
- dry-run が期待通りの日時を出力する
- 1件以上の予約保存が成功する
- 失敗時スクリーンショットが残る
- 認証異常時に停止する
- 16件超で翌日に繰り越す
