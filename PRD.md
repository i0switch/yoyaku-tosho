# PRD

## プロダクト名
X Scheduler Builder

## 背景
ユーザーは、複数の投稿文を事前に用意しておき、それらをXの公式予約投稿としてまとめて保存したい。
手作業で1件ずつ本文と日時を入力するのは時間がかかるため、この作業をローカル自動化したい。

## ゴール
- 投稿文をまとめて読み込める
- 指定した開始日から、7:00〜22:00の各時刻に順番に割り当てられる
- X公式Web UIに対して予約投稿として保存できる
- 投稿そのものは行わない
- 単一アカウント運用に限定する

## 非ゴール
- 投稿の自動公開管理
- エンゲージメント操作
- マルチアカウント運用
- サーバー常駐型SaaS
- スマホアプリ化
- ブラウザ拡張単体での完結

## ユーザーストーリー
### US-01
ユーザーとして、CSVに複数の投稿文を書きたい。
そうすることで、手入力を減らしたい。

### US-02
ユーザーとして、投稿文を7:00〜22:00の1時間間隔で自動割当したい。
そうすることで、毎回日時を考えずに済ませたい。

### US-03
ユーザーとして、X公式の予約投稿画面に自動で流し込みたい。
そうすることで、投稿準備作業を短時間で終わらせたい。

### US-04
ユーザーとして、異常が起きたら止まってほしい。
そうすることで、誤動作のまま大量処理されるのを防ぎたい。

## 機能要件
### F-01 データ入力
- mdを読み込めること


### F-02 スケジュール生成
- デフォルト枠は 07:00〜22:00
- 間隔は 60分
- 同日枠が埋まったら翌日に繰り越す
- 既に過ぎた時刻はスキップする
- タイムゾーンは Asia/Tokyo を前提とする

### F-03 予約投稿保存
- X の投稿作成画面を開く
- 投稿文を入力する
- 任意で画像を添付する
- 予約日時を入力する
- 予約保存する
- 保存結果をログへ残す

### F-04 実行制御
- dry-run モードを持つ
- max 件数指定を持つ
- 開始オフセット指定を持つ
- 途中エラー時は停止するモードを持つ
- 途中エラー時も続行するモードを持つ

### F-05 ログ
- 成功 / 失敗 / スキップを記録する
- 失敗時のスクリーンショットを保存する
- 実行サマリーを表示する

## 非機能要件
- Windows / macOS で動作可能
- Node.js 20+ を前提
- CLI 操作で完結
- ローカル保存のみ
- 可観測性が高いこと
- 失敗時に原因を追えること

## 制約
- XのUI変更により壊れる可能性がある
- 認証画面が出た場合は自動突破しない
- CAPTCHA 対応は行わない
- ログイン自体は原則手動初期化とする

## MVP定義
以下が揃えば MVP 完了とする。
1. CSV 読み込み
2. 予約枠自動生成
3. X で 1件以上予約保存
4. ログとスクリーンショット
5. dry-run
